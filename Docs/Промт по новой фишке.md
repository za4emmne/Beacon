 У тебя есть актуальная документация:  
`SECH_OVERVIEW-2.md`, `SECH_RULES-4.md`, `SECH_ROADMAP-3.md`, `SECH_WEAPONS-5.md`, `SECH_HORDE_BEHAVIOR.md`.  
Игра — survival roguelite в духе Vampire Survivors: Unity 2022, топ‑даун, система волн через `WaveSystem`, спавн врагов через `EnemiesGenerator`, прогресс и сохранения через `GameDataManager`, DI — Zenject.

Нужно спроектировать и частично реализовать **механику союзников (мини‑отряд)** и новый **старт забега**:

- После загрузки игры и появления игрока на карте:
    
    - враги ещё **не** генерируются;
        
    - таймер волны/забега **не** запускается;
        
    - открывается окно выбора союзника (или нескольких).
        
- Только после выбора союзника/союзников:
    
    - начинается забег: стартует `WaveSystem`, включается спавн врагов и таймер.

## 1. Общая концепция союзников

Нужна система **боевых союзников**, которые:

- бегают рядом с игроком;
    
- автоматически атакуют врагов (простое AI);
    
- могут быть несколько штук одновременно;
    
- выбираются в **отдельном окне в начале забега**, до старта волн.
    

Союзники должны хорошо ложиться на существующую архитектуру:

- данные через ScriptableObject (по аналогии с `CharacterData`, `WeaponData`, `EnemyData`);
    
- логика движения и атаки — отдельные компоненты, похожие на `EnemyMovement` и `WeaponController`, но под “наш” отряд;
    
- интеграция с `GameDataManager` (мета‑прогрессия: какие союзники открыты) и с UI (новое окно перед стартом волны).
    

---

## 2. Данные: ScriptableObject для союзника

Спроектируй `AllyData : ScriptableObject` c полями:

- `id` (string или enum);
    
- `displayNameKey`, `descriptionKey` — ключи для локализации;
    
- `icon` (Sprite — для UI);
    
- `prefab` (GameObject — сам союзник);
    
- базовые боевые параметры:
    
    - `maxHealth`,
        
    - `moveSpeed`,
        
    - `damage`,
        
    - `attackDelay`,
        
    - `attackRange`,
        
    - тип атаки: ближний/дальний (enum);
        
- прогресс/разблокировка:
    
    - `price` (монеты для покупки, если надо);
        
    - `isDefault` (доступен с начала);
        
    - опционально — требования по уровню/волне/ачивкам.
        

Нужен код этого ScriptableObject и пример того, где хранить ассеты (`Assets/Data/Allies`).

## 3. Логика союзника (AllyController / AllyBehavior)

Сделай компонент для префаба союзника, например `AllyController`, который:

- принимает `AllyData` в методе `Initialize(AllyData data, Transform playerTransform)`;
    
- всегда держится поблизости от игрока (следует за ним, но не перекрывает спрайт);
    
- выбирает цель среди ближайших врагов (можно использовать `EnemiesGenerator.AllEnemies` или аналогичный список);
    
- в зависимости от типа атаки:
    
    - ближний: подбегает на расстояние удара и бьёт;
        
    - дальний: стоит в радиусе и стреляет/кастует снаряды (можно переиспользовать простую логику из оружия);
        
- использует примитивный separation, чтобы союзники не слипались друг с другом и с игроком (можно упростить идею из `SECH_HORDE_BEHAVIOR.md`).
    

Опиши и покажи C#‑код `AllyController` с:

- движением к целевой позиции рядом с игроком;
    
- поиском ближайшего врага в `attackRange`;
    
- циклом атаки с учётом `attackDelay`.

## 4. Менеджер союзников

Нужен `AlliesManager` (Singleton или через Zenject), который:

- хранит список доступных `AllyData` (через массив в инспекторе или ScriptableObject‑коллекцию);
    
- умеет по запросу создавать набор активных союзников вокруг игрока:
    
    - `SpawnAllies(List<AllyData> selectedAllies, Transform playerTransform)`;
        
- по смерти союзника может его удалять/респавнить (базовый вариант — без респавна).
    

Опиши реализацию:

- где лучше инициализировать `AlliesManager` (в связке с `GameManager` / DI);
    
- как он взаимодействует с `GameManager` при старте игры.

## 5. Новый старт забега: выбор союзника до WaveSystem

Сейчас (по документации) старт игры выглядит так:

- `GameManager.InitializeGame` → `WaveSystem.StartWave` и т.д.
    

Нужно изменить пайплайн:

1. После загрузки сцены и появления игрока:
    
    - `GameManager` **не** запускает волны сразу;
        
    - вместо этого вызывает UI‑окно выбора союзника.
        
2. UI‑окно (например, `UIAlliesSelection`):
    

- показывает список доступных союзников (карточки с иконкой, именем, описанием);
    
- отмечает уже купленных и ещё закрытых (можно использовать `GameDataManager` — аналогично персонажам/локациям);
    
- позволяет выбрать одного или нескольких союзников (желательно ограничение по слотов, параметр в `GameDataManager` или в самом UI);
    
- имеет кнопку “Начать бой”.
    

3. После подтверждения выбора:
    

- `UIAlliesSelection` передаёт список выбранных `AllyData` в `AlliesManager.SpawnAllies`;
    
- только после этого `GameManager` запускает:
    
    - `WaveSystem.StartWave`
        
    - таймер забега
        
    - спавн врагов (`EnemiesGenerator`).
        

Нужны:

- схема взаимодействия `GameManager`, `UIAlliesSelection`, `AlliesManager`, `WaveSystem`;
    
- C#‑псевдокод или реальный код для обновлённого `GameManager.InitializeGame`, который теперь ждёт окончания выбора союзников.

## 6. Множественные союзники

Сразу заложи поддержку **нескольких** союзников:

- UI должен позволять выбрать 1–N союзников (например, до 3);
    
- `AlliesManager` спавнит их вокруг игрока в разных позициях (по кругу или полукругом);
    
- `AllyController` использует separation, чтобы они не накладывались друг на друга;
    
- При желании можно добавить простую роль:
    
    - “танк” ближе к врагам,
        
    - “лучник/маг” держится чуть дальше от игрока;
        
    - но базовая реализация может быть одинаковой — просто несколько бойцов.
        

Опиши:

- как лучше задать максимальное количество союзников (поле в `GameDataManager`, константа, параметр сложности и т.п.);
    
- как UI должен вести себя, если союзников больше, чем слотов (например, выбор из списка в стиле CharacterShop).
